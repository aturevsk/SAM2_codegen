Key Technical Discoveries

Masked Unit Attention (MUA)

Transition blocks spatially downsample by 2× using pooled Q tokens. Q tokens are 2×2 maxpooled within each window (T→T/4), while K/V remain at full window resolution. No pre-attention LayerNorm — applied post-add before MLP.

Anti-Aliased Resize

Precomputed bicubic index/weight tables stored as model buffers: aa_idx_w[1024,4], aa_wt_w[1024,4] (width axis), aa_idx_h[1024,3], aa_wt_h[1024,3] (height axis). Saved as float32, must be cast to int64 for indexing.

Lifted Tensors

162 Hiera backbone weights stored as graph constants (c_sam2_lifted_tensor_0..161), distinct from named parameters (p_sam2_*) and buffers (b_*).

Mask Decoder

2-layer transformer: self-attn (8h, 256d) + cross-attn token↔image (8h, 128d internal). Hypernetwork MLPs produce per-pixel mask weights. Output: 4 masks + IoU scores + obj_score.

Key Bugs Found & Fixed

Bug 1: SIGBUS (exit 138)
aa_idx_w/h index buffers saved as float32 by extractor but read as int64 bytes → reads 2× past EOF.
Fix: Load float32, cast to int64 element-wise.
Bug 2: Wrong ds_proj weights
Transition block skip path passed QKV weights (tensors 20-21) instead of actual downsampling projection (tensors 18-19).
Fix: Added separate ds_w_s2/s3/s4 fields.
Bug 3: Stage 3 block count
Code assumed 7 regular blocks in stage 3; actual count is 6. Caused tensors 130-131 (ds_proj for S3→S4) to be loaded as regular block weights → wrong FPN features.
Fix: Loop bound <8 → <7.
Bug 4: Transition block MLP norm
Used blk.norm2 (offset 10-11 in block) for pre-MLP LayerNorm in transition blocks. Graph shows the pre-MLP norm is at offset 4-5 (blk.norm1).
Fix: Use blk.norm1 for MLP norm in transition blocks.
Bug 5: norm1.size for transitions
load_hiera_block set norm1.size = C_in always. For transition blocks, norm1 operates on C_out-dim features (post-attention-add).
Fix: norm1.size = is_transition ? C_out : C_in.
Remaining: Numerical accuracy
After all fixes, mask RMSE = 4.89 vs PyTorch (down from 2.77e20). IoU ~6 vs expected ~0.98. Backbone features are approximately structured but subtly wrong. Requires step-by-step intermediate value comparison.

Key Distinction from Toolbox Import

This approach uses loadPyTorchExportedProgram directly — the same pattern as the Easy/repvit_deploy project. It does NOT use importNetworkFromPyTorch (Deep Learning Toolbox) which requires network graph parsing. Instead, the .pt2 ExportedProgram is consumed directly by MATLAB Coder's MLIR/TOSA lowering path.

Entry Function: sam2_infer.m

function [masks, iou_scores] = sam2_infer(image_hwc_u8, point_coords, point_labels) %#codegen % SAM2_INFER Run SAM2 inference via MATLAB Coder Support Package for PyTorch. % Uses loadPyTorchExportedProgram (NOT importNetworkFromPyTorch) persistent model; if isempty(model) model = loadPyTorchExportedProgram('/path/to/sam2_1200x1800.pt2'); end outputs = invoke(model, image_hwc_u8, point_coords, point_labels); masks = outputs{1}; iou_scores = outputs{2}; end
%% run_codegen.m — Invoke codegen cfg = coder.config('lib', 'ecoder', true); cfg.TargetLang = 'C++'; cfg.GenCodeOnly = true; cfg.EnableOpenMP = false; dlcfg = coder.DeepLearningConfig('none'); cfg.DeepLearningConfig = dlcfg; img_type = coder.typeof(uint8(0), [1200, 1800, 3], [false false false]); coords_type = coder.typeof(single(0), [1, 2], [false false]); labels_type = coder.typeof(int32(0), [1], [false]); codegen -config cfg sam2_infer ... -args {img_type, coords_type, labels_type} ... -d outDir -report
